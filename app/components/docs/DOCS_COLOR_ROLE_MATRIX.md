# Docs Color Role Matrix

Use these semantic text roles across docs UI to keep hierarchy consistent.

- Headlines (`h1`-`h4`): `color-pureWhite`
- Body / explanatory copy: `color-sand-11`
- Meta / labels / helper tags (step labels, small mono labels): `color-sand-8`
- Secondary / de-emphasis text: `color-sand-10`
- Functional accents (status, CTA emphasis, brand links): keep accent token usage

## Notes

- Prefer semantic tokens over hardcoded text colors.
- Keep visual card treatments unchanged unless readability requires a local exception.
- For light cards (for example Step 03 in Theme Generation), preserve contrast-safe dark text styles.
